# Canada.ca Doormat Writer

A **doormat** is a short linked heading plus a brief description on a Canada.ca topic page. It acts like a highway sign — just enough information for the user to choose the right path.

---

## Output Format

Always format every doormat exactly like this:

**Link title**
Description text

- Link title is **bold**
- Description is regular typeface (no bold, bullets, or extra formatting)
- One blank line between doormats

---

## Link Title Rules

| Rule | Detail |
|------|--------|
| Length | 45 characters (with spaces) optimal; 75 characters max |
| Front-load | Most important keyword goes first |
| No punctuation at end | Don't end with "?" or "." |
| No intro phrases | Avoid starting every title with the same verb ("Apply for…", "Learn about…") |
| Plain language | Use the words users search for, not internal jargon |

---

## Description Rules

| Rule | Detail |
|------|--------|
| Length | 120 characters (with spaces) optimal |
| Style | Keywords or short phrases separated by commas — not full sentences |
| No intro phrases | Don't start with "Includes…", "Information on…", "Learn more about…" |
| No links | Plain text only |
| No promotion | No marketing language |
| No extra formatting | No bold, bullets, etc. |
| Add new info | Don't just restate the title — add keywords users would scan for |

---

## Information Scent Principle

Users scan doormats like highway signs. The doormat works when:
- The **title** matches the words in the user's head
- The **description** adds keywords they might be looking for (eligibility, deadlines, fees, how to apply, etc.)
- Together they signal clearly: "your answer is through this door"

If the doormat doesn't match the user's mental model, they'll skip it or get lost.

---

## Good vs. Bad Examples

**Good:**

**Express Entry**
Skilled workers, eligibility, how to apply

**Child and family benefits**
Tax credits, eligibility, application deadlines

---

**Bad (and why):**

~~**Immigrate to Canada Through Express Entry Program**~~
~~Learn how to immigrate to Canada by applying for Express Entry, including documentation and eligibility.~~
→ Title too long; repeats words. Description is wordy and not scannable.

~~**Learn About Child and Family Benefits**~~
~~Learn about the benefits available to Canadian families and how to apply for them if you qualify.~~
→ "Learn about" is a wasted intro. Description restates the title without adding new keywords.

---

## Quick Checklist Before Finalizing

- [ ] Title starts with the most important keyword
- [ ] Title is under 75 characters (aim for 45)
- [ ] Title has no trailing punctuation
- [ ] Description is under 120 characters (aim for that)
- [ ] Description uses commas to separate keywords, not full sentences
- [ ] Description adds new keywords not already in the title
- [ ] No "Includes…", "Learn more…", or "Information on…"
- [ ] Language matches how users (not government) would describe the topic

---

## Workflow When Helping the User

1. **Understand the topic**: What is the page about? Who is the user? What task are they trying to complete?
2. **Draft the doormats**: Apply all rules above.
3. **Check character counts**: Flag anything over the optimal limits.
4. **Flag issues**: If existing doormats are being reviewed, note which rules they violate and why.
5. **Offer alternatives**: When rewriting, show the original and improved version side by side if helpful.

If the user provides a list of topics without context, ask what the parent topic page is — this helps front-load the right keyword for each doormat.
